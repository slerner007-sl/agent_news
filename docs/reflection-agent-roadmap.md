# Агент рефлексии — roadmap

## Текущее состояние

`insights.py` работает в один проход: новости текущего `run_id` → LLM → рекомендации.
Это реактивная классификация, а не рефлексия. Агент не смотрит на прошлые дни,
не читает фидбек, не ищет тренды.

## Чего не хватает

### 1. Темпоральный анализ
Смотреть на новости за N дней, находить повторяющиеся темы, нарастающие тренды,
цепочки событий. Пример: «компания X упоминалась 3 раза за неделю в контексте
долгов → это паттерн, а не единичное событие».

### 2. Цикл обратной связи
Агент не читает `insight_feedback`. Если менеджеры массово ставят «бесполезно»
на определённый тип инсайтов — агент этого не знает и продолжает их генерировать.
Нужно: подавать статистику фидбека в промпт, корректировать пороги и приоритеты.

### 3. Кросс-ГОСБ паттерны
Один холдинг может фигурировать в новостях разных регионов. Сейчас каждый ГОСБ
обрабатывается изолированно. Нужен слой, который видит картину целиком.

### 4. Мета-рефлексия
Агент не анализирует качество своих прошлых инсайтов. Нужен цикл:
«Из 50 инсайтов за неделю 40 были boring → пересмотреть стратегию генерации».

### 5. Обнаружение новых зависимостей
Сейчас агент классифицирует по заданным типам. Нет этапа поиска неожиданных
связей в данных за длительный период.

### 6. Persistent memory
Workspace агента имеет `memory/`, но `insights.py` его не читает и не пишет.
Между запусками агент «забывает» всё.

## Предлагаемая архитектура циклов

### Цикл 1: Ежедневный (расширить текущий)
- Новости сегодня + контекст последних 3–7 дней → инсайты
- Подавать в промпт: статистику фидбека за последнюю неделю, предыдущие инсайты

### Цикл 2: Еженедельный (новый)
- Входные данные: инсайты за неделю + фидбек + тренды новостей
- Выходные данные: мета-инсайты, корректировки стратегии
- Примеры: «За неделю 5 новостей по теме X → формируется тренд»,
  «Менеджеры игнорируют competitor_signal → снизить приоритет»

### Цикл 3: Стратегический / месячный (новый)
- Входные данные: данные за месяц, все инсайты и фидбек
- Выходные данные: новые зависимости, скрытые паттерны, обновление memory
- Примеры: «Клиент Y упоминается в связке с регуляторными рисками чаще других»,
  «Метрика Z коррелирует с типом новостей K»
